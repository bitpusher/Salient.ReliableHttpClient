Public Methods
 
Name 
Description 
 
OnActivityWarning 
Notification that the connection is stalled (i.e. nothing has been received on the connection for more than the "probeWarningMillis" timeout configured on the ConnectionInfo) or that the connection is working again after being notified as stalled.
If the connection is notified as stalled and nothing is received for more than the "probeTimeoutMillis" timeout configured on the ConnectionInfo, then the connection is forcibly closed with an onFailure notification.
 
 
OnClose 
Notification of the closure of the Connection. It may be caused by a CloseConnection call or by an unrecoverable error. There is no guarantee that calls to this listener will not be followed by further calls on some listener related to a table subscribed within the connection. Such extra calls should be ignored.  
 
OnConnectionEstablished 
Notification of the opening of the connection to the Server. 
 
OnDataError 
Notification that a block of data received from the Server could not be correctly interpreted. The implementation may consider this as a severe problem and open a new connection. In normal operation, this should not happen.  
 
OnEnd 
Notification of the closure of the Connection because of a Server decision. Upon this notification, it is possible to perform recovery actions, such as trying a reconnection; however, it may be advisable not to try a new connection immediately.  
 
OnFailure 
This is the overview for the OnFailure method overload. 
 
OnNewBytes 
Notification of the reception of some data bytes from the Server, for statistical purposes.  
 
OnSessionStarted 
Notification that a session has been started by the Server on the connection.
